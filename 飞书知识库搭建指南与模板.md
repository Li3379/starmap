### 模板B：个人任务台账（飞书多维表格）

在飞书建"多维表格"，字段配置：

| 字段名 | 字段类型 | 配置说明 |
|--------|---------|---------|
| 任务ID | 文本 | 如 R3-W2-01（角色-周-序号）|
| 任务描述 | 文本 | 具体做什么 |
| 阶段 | 单选 | 阶段0 / 阶段1 / ... / 阶段7 |
| 周次 | 单选 | W1 / W2 / ... / W15 |
| 状态 | 单选 | ⬜未开始 / 🔄进行中 / ✅已完成 / 🔴阻塞 |
| 优先级 | 单选 | P0关键 / P1高 / P2中 / P3低 |
| 预计完成 | 日期 | |
| 实际完成 | 日期 | |
| 依赖项 | 文本 | 依赖谁的什么任务（如"依赖R3契约签字"）|
| 交付物 | 文本 | 产出什么文件/功能 |
| 阻塞说明 | 文本 | 阻塞原因、需要谁协助 |
| 进度 | 进度条 | 0-100% |

**视图建议**：
- 视图1"看板视图"：按"状态"分组
- 视图2"甘特图"：按"周次"排列
- 视图3"本周任务"：筛选"周次=当前周"

## 三、飞书机器人/自动化（可选，进阶）

如果团队用飞书机器人，可以配置：

1. **GitHub → 飞书通知**：PR 合并、CI 失败自动推送到群
2. **每日提醒**：每天 21:00 提醒全员更新看板
3. **周报自动汇总**：从看板拉取本周完成项，生成周报草稿

---

## 四、同步规则（重要）

1. **看板是任务源**：GitHub Projects 看板是唯一任务来源，飞书只做汇总展示
2. **代码在 GitHub**：飞书不放代码，只放文档和讨论
3. **决策落文档**：群里讨论的决策必须同步到飞书"04-决策记录"
4. **周报每周五**：R8 在迭代评审后 2 小时内更新飞书周报
5. **个人台账每人维护**：每个人每天更新自己的飞书台账状态
6. **重大变更群通知**：契约变更、模型变更，飞书更新 + 群@相关人 + GitHub Issue
